# Config Drift Watcher

[English](README.md) · **Русский**

Плагин для Obsidian, который обнаруживает, когда реальная конфигурация сервера разошлась с тем, что задокументировано в твоих заметках — только чтение, по SFTP.

## Проблема

Документация по инфраструктуре устаревает. Ты записываешь, как должен быть настроен сервер, а потом ребут, ручная правка или чужой скрипт меняют реальный файл — и заметки тихо становятся неверными. Узнать, что документация устарела, обычно получается только когда что-то ломается в проде.

## Что делает

Ты размечаешь код-блок `drift` в заметке псевдонимом сервера и путём к удалённому файлу. Плагин подключается к этому серверу по SSH в режиме только чтения, забирает текущее содержимое файла по SFTP и сравнивает его построчно с код-блоком. Если совпадает — блок помечается как синхронизированный. Если различается — плагин показывает, какие строки есть только в заметке, а какие только на реальном сервере, и саму заметку не трогает.

Плагин **никогда не выполняет команды** и **никогда не пишет на удалённый сервер**. Только чтение.

## Использование

Размечай код-блок `drift`. Первая строка — `target: alias:/абсолютный/путь`; всё после неё — задокументированное содержимое, которое сравнивается с удалённым файлом:

````markdown
```drift
target: gammastack-stfox:/etc/network/interfaces
auto eth0
iface eth0 inet dhcp
iface eth0 inet6 manual
```
````

> **Важно:** открывающий ` ```drift ` должен начинаться **с начала строки** (без пробелов перед кавычками). Блок с отступом в режиме чтения рендерится как обычный Markdown, и плагин его не обработает.

Блок отображается карточкой со статус-бейджем и двумя действиями:

- **Check drift** — читает текущий удалённый файл и сравнивает с содержимым блока, не меняя заметку. Различия показываются инлайн под блоком.
- **Snapshot from server** — читает текущий удалённый файл и заменяет им содержимое блока. Так удобно снять базовый слепок, а потом отредактировать блок вручную, если нужно задокументировать *намеренное* состояние, а не текущее. Если в блоке уже есть содержимое, действие сначала запросит подтверждение, чтобы ручная правка не была молча перезаписана.

### Статус-бейджи

| Бейдж | Значение |
| --- | --- |
| **Not checked** | Блок ещё не сверялся в этой сессии. |
| **Checking… / Reading…** | Идёт чтение. |
| **In sync** | Заметка и удалённый файл совпадают. |
| **Drift · N lines** | Различаются; инлайн-diff показывает, что изменилось. |
| **Error / Unknown alias / Invalid block** | Что-то пошло не так; сообщение объясняет причину. |

## Как работает сравнение

Сравнение — это простой построчный текстовый diff. Никакой интерпретации смысла конфига, никакого ИИ и никакого знания о конкретных форматах (nginx, sysctl, network interfaces и т.д. — всё считается простым текстом). Плагин настолько корректен, насколько корректно то, что ты написал в заметке: если задокументировать неправильное, плагин честно скажет, что неправильное не совпадает с реальностью.

В настройках есть две опции сравнения, чтобы убирать шум на реальных конфигах:

- **Ignore whitespace** — считать различия только в пробелах синхронизированными.
- **Ignore comments** — отбрасывать строки-комментарии (по настраиваемому списку префиксов, например `#`, `;`, `//`) перед сравнением.

Завершающий перевод строки в конце удалённого файла (которого нет у код-блока) никогда не считается дрифтом.

## Профили серверов

Учётные данные никогда не хранятся в заметке. Каждый псевдоним, использованный в код-блоке (например `gammastack-stfox`), ссылается на профиль подключения, который настраивается один раз в настройках плагина:

| Поле | Описание |
| --- | --- |
| **Alias** | Имя, на которое ссылаются заметки как `target: alias:/path`. |
| **Host** | IP-адрес или имя хоста сервера. |
| **Port** | SSH-порт (по умолчанию `22`). |
| **Username** | SSH-пользователь для подключения. |
| **Private key path** | Путь к приватному ключу на этой машине (`~` раскрывается). |
| **Key passphrase** | Только если приватный ключ зашифрован. |

Заметки ссылаются лишь на псевдоним — поэтому их безопасно синхронизировать, шарить и коммитить в систему контроля версий.

## Способ подключения

Удалённые файлы читаются по **SFTP**, а не выполнением shell-команд. Это осознанный выбор: чтение по SFTP может только прочитать файл и не может выполнить произвольный код на удалённом хосте, как бы ни был сформирован путь или псевдоним.

Аутентификация — **только по ключу**. Парольная аутентификация не поддерживается.

## Безопасность

Плагин подключается к удалённым серверам и работает с SSH-учётными данными. Прочитай это перед использованием.

- **Только чтение by design.** Плагин выполняет лишь SFTP-чтение. Он никогда не выполняет удалённую команду и никогда не пишет на сервер. Он не может применить фикс или что-либо изменить на хосте.
- **Учётные данные — в данных плагина, не в заметках.** Профили (host, username, путь к приватному ключу и опциональный passphrase) хранятся в `.obsidian/plugins/config-drift-watcher/data.json` внутри твоего хранилища. В заметках только псевдоним. Если ты синхронизируешь или публикуешь хранилище, помни, что `data.json` содержит эти данные подключения — исключи его (или папку `.obsidian` целиком) из всего публичного. **Passphrase ключа, если задан, хранится в `data.json` в открытом виде.**
- **Приватный ключ не покидает машину.** Плагин читает файл ключа с локального диска для аутентификации — ровно как команда `ssh`. Сам ключ в хранилище не копируется.
- **Верификация host key (доверие при первом подключении).** При первом подключении к профилю SHA-256-отпечаток host key сервера запоминается и закрепляется. При последующих подключениях изменившийся отпечаток прерывает чтение с предупреждением — это защищает от «человека посередине» или неожиданной смены сервера. **Оговорка:** доверие-при-первом-подключении не обнаружит перехват, который уже присутствует на самом первом подключении — закрепи отпечаток заранее, если нужна эта гарантия (см. ниже).
- **Только десктоп.** Плагину нужны Node-API (SSH и доступ к локальным файлам), на Obsidian mobile он не работает.

### Верификация host key

По умолчанию каждый профиль запоминает и закрепляет host key сервера при первом подключении и показывает отпечаток в уведомлении. Можно также проверить или закрепить его вручную:

- Формат отпечатка совпадает с OpenSSH, так что его можно сравнить с:

  ```bash
  ssh-keygen -lf <(ssh-keyscan -t ed25519 your.host 2>/dev/null)
  ```

- Чтобы **жёстко закрепить с самого начала**, вставь известный отпечаток `SHA256:…` в поле профиля *Host key fingerprint* до первого подключения.
- Чтобы **переподтвердить** после легитимной смены сервера, нажми *Forget* рядом с этим полем; следующее подключение запомнит новый ключ.

## Ограничения

- Один файл на код-блок. Дрифт по целым каталогам в текущей версии не поддерживается.
- Только чтение by design. Плагин никогда не применит фикс на сервере автоматически.
- Требуется аутентификация по SSH-ключу. Пароль не поддерживается.
- Без расписания. Проверки запускаются по требованию, а не по таймеру, в текущей версии.

## Планы

- Интеграция с файлом `known_hosts` (сейчас отпечатки закрепляются на уровне профиля)
- Опциональные фоновые проверки по расписанию с уведомлением о дрифте
- Слежение за каталогами и несколькими файлами
- Правила игнора в diff на уровне блока (пробелы, комментарии, порядок)
- Опциональная интеграция с LLM для подсказки команды-фикса при найденном дрифте, выключена по умолчанию

## Установка

### Вручную

Пока не опубликовано в каталоге community-плагинов. До этого — ставь вручную:

1. Скачай `main.js`, `manifest.json` и `styles.css` из последнего релиза.
2. Создай папку `config-drift-watcher` внутри `<vault>/.obsidian/plugins/`.
3. Положи туда эти три файла.
4. Включи плагин в разделе Community plugins настроек Obsidian.

### BRAT (бета)

Если используешь плагин [BRAT](https://github.com/TfTHacker/obsidian42-brat), добавь этот репозиторий (`rixct/config-drift-watcher`), чтобы поставить и автообновлять бету до попадания в каталог.

## Разработка

```bash
npm install        # установить зависимости
npm run dev        # сборка в режиме watch
npm run build      # проверка типов + продакшн-бандл
npm test           # юнит-тесты (parser, diff, snapshot, hostkey)
```

Плагин бандлит `ssh2` и `diff` в один `main.js` через esbuild.
